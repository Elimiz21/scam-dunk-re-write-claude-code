6. Effectiveness — my view

Bottom line: the product is currently running on one cylinder of four, and even that cylinder is misfiring.

Coverage. The marketed "Four-Layer Detection Engine" is, in practice, Layer 1 only — and only when a user clicks Scan. There has been no daily sweep of the ~6,500-ticker universe for more than two months. From a product-claims perspective this is a significant gap between the white-paper / blog-post description and what the system is actually doing.

Precision. Layer 1 in isolation is structurally a false-positive machine: when it ran (Feb–Mar), 15–24% of all listed stocks came out HIGH. That is not a usable signal — it's "every penny stock is risky", which is true but trivial. Layers 2–4 (anomaly detection, Random Forest, LSTM) exist specifically to suppress those false positives, and they aren't connected.

Social scan. It is the only thing earning its keep. 32 runs / 50 days, ~32k deduped mentions, ~6k promotional, and it is correctly surfacing high-conviction pump candidates like INVZW, AIXI, SELX, POM, SGBX. But:

  • It covers only 50 tickers per day, and the seed list looks stale (same names recur). With the main scan offline there is no fresh source of "today's suspicious stocks" feeding it.
  • Reddit native scraping has failed on 100% of runs for at least 50 days; Discord returns 0 every day; X/Twitter has no direct scraper. The "six platform" claim is effectively "StockTwits + Google search + YouTube" — three sources.
  • 0 of 32 runs were COMPLETED — every run is either PARTIAL or TIMED_OUT, so the dashboard's success metric is always amber/red.

Alerts & watchlists. Risk-change alerting (StockRiskAlert) hasn't fired since 9 March. The promoted-stock watchlist hasn't been added to since 24 February. So even when the social scan does find a fresh pump, nothing downstream picks it up automatically.

Severity. I'd rate the current effectiveness of the daily-scan stack at roughly 20–25% of the designed value. The defensible part is the social-mention dataset (which is genuinely large and contains real signal). The undefensible part is that the AI engine and the bulk daily evaluation — the headline features — are silent.

The three things to fix, in order

  1. Bring the Python AI backend online and point AI_BACKEND_URL at it in production. Without this, Layers 2–4 cannot ever run, and Layer 1 will keep generating ~15% HIGH-risk noise. The code is all there in python_ai/; this is a deploy/wiring problem, not a build problem.
  2. Restart the daily Layer-1 sweep and the alerts pipeline. Whatever cron / Railway worker was running through 16 March stopped — re-enable it, and re-instate the StockRiskAlert generator that drove the 22,084 historical alerts. Even L1-only is more useful when it runs daily and feeds risk-change deltas.
  3. Fix the social-scan platform mix. Get a working Reddit path (OAuth, not the public JSON which is permanently 403), wire up a real X/Twitter scraper (it has the highest promotional density of any source we have), and either fix or remove the Discord bot. Then expand the seed list past the static 50 tickers — feed it from yesterday's L1 HIGH-risk output.
